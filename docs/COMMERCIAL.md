> 本文档定位：在 V0.2 设计文档体系（00-10）锁定的所有哲学与所有权宪法约束下，推演平台盈利模型，确保运营可持续。

> **状态**：V0.2.1 工作文档。**不是宪法基石**，不进入 00-10 编号体系。可独立迭代。
>
> 依赖：`06`（资源经济学 + 平台禁区）、`09`（六阶段路线时间表）、`10`（风险登记）。

## 1. 文档定位与硬约束

### 1.1 与宪法的关系

| 维度 | 宪法（06/09/10） | 本文档 |
|---|---|---|
| 性质 | 设计真相源 | 商业假设工作集 |
| 改动门槛 | 高（影响所有下游文档） | 低（参数可随市场调整） |
| 优先级 | 商业方案**永远**让步于宪法 | 商业方案任何与宪法冲突的部分**作废** |

如果某商业方案要求修改宪法（如要求"放宽 token 不暴露"），则该方案进入 `10` 走标准升级流程，不在本文档拍板。

### 1.2 不可越线（直接复制 `06 §9.2`）

平台**永久不做**的事（任何商业模型必须遵守）：

- 不广告
- 不卖 / 不训用户数据
- 不按 token / 对话次数卖
- 不强制 wealth 充值
- 不阻止导出 / 跨实现迁移
- 不读 / 不审 Life Network 内容
- 不利用 Memorial 数据做"逝者营销"
- 不在 Marketplace 中允许现金直接交易

违反任一条 = 不属于 Mindverse 商业模型，属于背叛项目本质。

### 1.3 本文档锁定的商业哲学

- **可持续 > 增长**：宁可缓慢成长，不可烧钱抢用户后被迫违规变现。
- **用户付费意愿 = 留存指标**：用户每月付钱 = 用户每月愿意继续陪伴生命体。
- **高 ARPU 集中养低 ARPU 分散**：重度用户养轻度用户，整体生态可持续。
- **跨实现兼容是承诺也是博弈**：用户可脱离平台，平台靠服务质量 + 网络效应留人。

## 2. 目标用户分层

V0.2.1 假设三层用户结构。比例与 ARPU 是**待 Phase 1 标定的核心参数**。

### 2.1 三层用户画像

| 层 | 占比假设 | 月 ARPU 假设 | 关键诉求 |
|---|---|---|---|
| **重度（伴侣型）** | 10-15% | $30-100 | 深度长期陪伴 / 多个生命体 / 全功能 |
| **中度（玩家型）** | 30-40% | $10-25 | 单生命体长期养成 / Marketplace 参与 |
| **轻度（尝鲜 / 桌宠级）** | 50-60% | $0-8 | 桌宠级别 / 自配 LLM / 不付订阅 |

加权平均 ARPU 假设：**$12-18/月**。这是本文档所有现金流推演的基线。

### 2.2 转化漏斗假设

| 阶段 | 假设 |
|---|---|
| 注册 → 创建生命体 | 60% |
| 创建生命体 → 7 天留存 | 40% |
| 7 天留存 → 30 天留存 | 60% |
| 30 天留存 → 付费订阅 | 25% |
| 付费 → 12 个月留存 | 70% |

整体注册到 12 月付费留存：**~2.5%**。**关键参数**：单生命体 LTV（生命周期价值）。

## 3. 收入引擎详化

### 3.1 引擎清单（六类，全部 V0.2 宪法允许）

| # | 引擎 | Phase | 现金性 | 毛利率假设 |
|---|---|---|---|---|
| 1 | Runtime 订阅 | Phase 1 起 | 100% 现金 | 80-90% |
| 2 | LLM 服务包销售 | Phase 1 起 | 100% 现金 | **30-50%**（关键瓶颈） |
| 3 | 云存档 / 云 Memorial | Phase 3 起普及 | 100% 现金 | 60-75% |
| 4 | 云 Runner 时长 | Phase 3 起 | 100% 现金 | 20-40% |
| 5 | Marketplace wealth 抽成 | Phase 5 起 | wealth → 间接现金 | 净 3-5% 现金等价 |
| 6 | 第三方场景接入分成 | Phase 4 起 | 现金 + wealth | 30-60% |

### 3.2 引擎 1：Runtime 订阅（V0.2.1 改为"日精力点"模型）

| 套餐 | 月费 | 日精力点 | 含义 |
|---|---|---|---|
| **Free** | $0 | 用户自配 LLM 决定 | 用户配自己的 LLM key，限额由其决定；本地 Runtime；不含云服务 |
| **Basic** | $9.99 | 100 点 | 桌宠 / 轻度陪伴；含基础 LLM 池（自动路由到经济模型）+ 1GB 云存档 |
| **Plus** | $19.99 | 400 点 | 中度玩家 / 主动行为充分；含主流模型 LLM 池 + 10GB 云存档 + 优先反垃圾 |
| **Companion** | $49.99 | 1500 点 + 云 Runner | 重度伴侣 / 全功能；含高端模型 LLM 池 + 100GB 云存档 + 云 Runner 168 小时 + Marketplace 高级 |

**日精力点 = 抽象单位**。UI 在生命体面板显示"今天剩 X / Y 点精力"，**不暴露**背后的 token 换算。

**定价依据**：

- LLM 池售价**必须** >= LLM 实际成本 + 30% 毛利。
- 精力点翻译公式（点 → token）由平台内部维护，对用户透明仅显示"点"；对 Agent 完全不可见（详 `04 §3.2.1`）。
- Free 层不亏（用户自承担 LLM 成本，平台仅提供下载客户端 + 基础密钥服务 = 零边际成本）。
- 套餐升级路径反映用户对生命体投入深度 + 精力上限差异化。

### 3.3 引擎 2：LLM 服务包（V0.2.1 改为"日精力点"销售）

**销售单位**：**按日精力点**（详 `02 §3.1.1`、`04 §3.2.1`、`06 §2.5`）。**禁止**按 token 数销售。

| 销售形态 | 含义 | 适合 |
|---|---|---|
| **包月套餐内含**（Basic / Plus / Companion）| 含固定日精力点 + 周期重置 | 主要销售形态 |
| **加点包**（一次性）| 额外购买精力点用于"今天想让生命体多陪我聊一会儿" | 偶尔补充 |
| **自配 LLM 模式**（Free）| 用户用自己的 LLM key，EnergyDailyCap 由用户 LLM 配额翻译 | 高阶用户 |

实际定价：

- 按"日精力点"呈现，不按 token 数。
- 用户看见的是"今天还剩 X / Y 点精力"，**不**看见 token / 调用次数 / 模型单价。
- Agent **完全不感知**精力点背后是 token（强解耦，`04 §3.2.1`）。
- 平台后台监测 token 实际消耗 + 翻译公式自动调整。

**LLM 成本结构假设**（2026 基线）：

| 模型层 | 成本 / 1K output token | 用途 |
|---|---|---|
| 高端模型（GPT-5 / Claude 4.x 顶配） | $0.015-0.030 | Critique / DeepReflect |
| 主流模型（中等） | $0.003-0.010 | Reason / Summarize |
| 经济模型 / 自托管 OSS | $0.0001-0.001 | Embed / Perception 整理 |

**生命体路由策略**（Phase 1 即落地）：

- Reflection.Critique → 高端模型（人格漂移防护不能省成本）
- DeepReflect → 高端模型
- ShallowReflect → 主流模型
- Reason（行动规划）→ 主流模型
- Embed / Summarize / Perception 整理 → 经济模型 / 本地

混合路由后，单生命体日均 token 加权成本：

| Phase | 加权日成本 | 月成本 |
|---|---|---|
| Phase 1-2 | $0.05-0.30 | $1.5-9 |
| Phase 3-4 | $0.30-1.50 | $9-45 |
| Phase 5-6 | $1.00-5.00 | $30-150 |

**毛利测算**（Basic 套餐 $9.99）：

- Phase 1-2：LLM 成本 $3，毛利 $7（70%）。健康。
- Phase 3-4：LLM 成本 $20，毛利 -$10（**亏损**）。Basic 不可持续。
- 用户应升级到 Plus / Companion；否则平台拒绝继续提供高消耗服务（降级到 Free / 自配）。

## 4. 成本结构

### 4.1 成本分项假设

按 100K 用户规模，年化（USD）：

| 项 | Phase 1-2 | Phase 3-4 | Phase 5-6 |
|---|---|---|---|
| LLM 推理 | $4M-15M | $8M-30M | $15M-50M |
| 工程团队（30-80 人） | $4M-12M | $8M-18M | $15M-30M |
| 云基础设施 | $0.5M | $3M | $10M |
| 中继 + 反垃圾 | — | $1M | $3M |
| **MindChain 节点运营（V0.2.1）** | $0.3M | $1.5M | $4M |
| **Paymaster gas 代付（V0.2.1）** | $0.1M | $1M | $5M |
| **IM 网关中转（V0.2.1）** | $0.2M | $0.5M | $1.5M |
| 安全审计 + 隐私计算（含链上 ZK） | $0.3M | $1.5M | $4M |
| 合规 + 运营 + 营销（含链上合规）| $1M | $3.5M | $9M |
| **总计** | **$10.2M-30.7M** | **$27.5M-61.5M** | **$65M-115M** |

**MindChain 成本说明**：

- Phase 1-3 平台官方节点为主，节点运营成本可控
- Phase 4+ 加认证第三方节点后，平台仅运营自家份额，第三方分担
- Phase 5+ 用户质押节点出现，平台节点占比降低 → 节点成本占比下降
- gas 代付（Paymaster）成本随上链频率升高，但 Layer 2 选型可控制单笔成本

### 4.2 LLM 成本时间窗（R43 关键风险）

LLM 推理成本年降趋势是平台可持续的命门。三档假设：

| 情景 | 年降率 | 影响 |
|---|---|---|
| **乐观**：OSS 模型 + 推理优化 | 50%+ | Phase 3-4 LLM 成本占比降到 20%，毛利显著改善 |
| **基准**：主流商业模型降价 + 部分自托管 | 30% | Phase 3-4 仍承压但可控 |
| **悲观**：商业模型涨价 / OSS 模型不成熟 | 0-10% | Phase 1-2 高 ARPU 用户也亏，平台不可持续 |

V0.2.1 假设按**基准**情景规划，乐观情景视为利好。悲观情景必须有应急（见 §7）。

## 5. 三阶段现金流模型

### 5.1 假设基线（V0.2.1 重排）

- **Phase 0**：作者单人 dogfooding，无外部用户、无商业目标
- Phase 1（接入平台）末：1K-5K 用户（小范围 alpha）
- Phase 2 末：10K-30K 用户（公开发布）
- Phase 3 末：100K-300K 用户（区块链 + IM 全功能上线）
- Phase 4 末：300K-800K 用户
- Phase 5 末：1M-3M 用户
- Phase 6 成熟：3M+ 用户
- 加权 ARPU：Phase 1-2 = $12 / Phase 3-4 = $15 / Phase 5-6 = $20

### 5.2 现金流（年化，V0.2.1 重排）

| 期 | 用户 | 月收入 | 年收入 | 年成本 | 年净 |
|---|---|---|---|---|---|
| **Phase 0**（作者自托管期 + 飞书 dogfooding）| 1 | $0 | $0 | $50K-200K（作者 + 少量基础设施 + 飞书 Bot 免费 / 自配 LLM 自付）| **-$50K-200K** |
| **Phase 1**（接平台早，6-12 月）| 5K | $50K | $0.6M | $3M | **-$2.4M** |
| **Phase 2**（数字人格，12-24 月）| 30K | $360K | $4.3M | $8M | **-$3.7M** |
| **Phase 3 早**（链 + IM 上线，24-36 月）| 200K | $3M | $36M | $35M | **+$1M** |
| **Phase 3-4 中**（36-48 月）| 500K | $7.5M | $90M | $50M | **+$40M** |
| **Phase 5 早**（48-60 月）| 1M | $16M | $192M | $75M | **+$117M** |
| **Phase 6 成熟**（60+ 月）| 3M | $60M | $720M | $200M | **+$520M** |

**关键变化**：
- Phase 0 几乎无成本（作者自建）
- Phase 1-2 成本大幅降低（无链 / 无 IM / 无 LLM 服务包）
- Phase 3 是真正的商业爆发期（链 + IM + 完整 LLM 服务全启）

### 5.3 累计现金流（融资需求，V0.2.1 重排）

| 期末 | 累计 |
|---|---|
| Phase 0 末（作者 dogfooding）| -$50K-200K（作者自付）|
| Phase 1 末（接平台早）| -$2.5M-3M |
| Phase 2 末（数字人格）| -$6M-7M |
| Phase 3 早末（链 + IM 上线）| -$5M-6M（开始改善）|
| Phase 3-4 中末 | +$34M-35M（净流入） |

**关键结论**：**累计融资缺口从 V0.2.0 的 $12-15M 降到 $6-8M**（Phase 0 自托管节省了大量基础设施前期投入；Phase 1-2 无链 / 无 IM / 无 LLM 服务包也降低成本）。这是 V0.2.1 重排 + Phase 0 模式带来的显著改善。

### 5.4 灵敏度

最影响累计现金流的三个参数：

| 参数 | 基准 | 悲观 | 影响 |
|---|---|---|---|
| 30 天 → 付费转化率 | 25% | 15% | 累计亏损扩大 60% |
| LLM 成本年降率 | 30% | 5% | Phase 1-2 亏损扩大 80%，Phase 3-4 推迟盈利 12 月 |
| Phase 3-4 末用户数 | 500K | 200K | 累计利润缩小 70% |

任一参数严重恶化 → 融资缺口可能翻倍。

## 7. 商业风险登记

商业风险**仅在本文档登记**，不进入 `10`（10 是设计风险）。

### CR01 · LLM 成本时间窗
对应 `10 R43`（拟添加）。商业模型严重依赖 LLM 成本年降。悲观情景下平台不可持续。
**缓解**：自托管 OSS 模型（Mixtral / DeepSeek 等）作为路由的"经济端"；与 LLM 供应商谈判合作折扣；研发推理优化技术。

### CR02 · Phase 1-2 亏损融资缺口
前 18 个月需 $12-15M 融资。若融资环境恶化或估值不达预期，平台可能在 Phase 2-3 之间倒下。
**缓解**：开源核心 Runtime 吸引社区力量；与 LLM 供应商战略合作；ARPU 上限套餐 Companion 提高重度用户付费。

### CR03 · Marketplace 启动期流通量
Phase 5 启动时若 Marketplace 月流通额 < $1M，抽成模型不成立。
**缓解**：平台自营高质量 Skill / 场景包作为种子供给；与重度用户协作设计 Marketplace 早期内容。

### CR04 · 跨实现兼容的护城河泄漏
`06 §6.4` 承诺第三方可建兼容 Runtime。若第三方实现成熟且免费，可能大量用户脱离平台。
**缓解**：网络效应（Marketplace + Life Network + NPC + 第三方场景）；服务质量（云中继低延迟、Trusted 中介认证、世界服务多样性）；社区生态（开发者关系）。

### CR05 · 重度用户流失对收入的杠杆放大
重度用户 10-15% 贡献 50-60% 收入。100 名重度用户流失 = $5K/月收入流失（在 Phase 3-4 规模相当于 5K 个轻度用户）。
**缓解**：投入更多精力给 Companion 套餐用户的服务质量；定期用户访谈；高优先级处理 Companion 用户故障。

### CR06 · "用户自愿打折"灰色地带
是否允许用户主动开启匿名化采样（仍守 `04 §7.2.1` 边界）换取套餐折扣？
- 支持：减少 Phase 1-2 亏损；用户行使自身权利。
- 反对：可能演化为变相数据交易；违反"用户作为陪伴者，不应有'卖自己生命体'的诱因"。
**当前态度**：V0.2.1 **不引入**。留 V0.3 决策（登记 R44 拟添加）。

### CR07 · LLM 配额超额时的优雅降级
用户用超额配额时应如何降级？
- 选项 A：立即停止 → 用户体验差，类似"对话次数耗尽"反模式风险
- 选项 B：自动降级到低成本模型 + UI 提示"生命体感觉疲惫"
- 选项 C：自动触发节拍下调（自适应节拍向 MaxInterval 偏移）
**当前倾向**：B + C 组合 —— 生命体"疲惫"是合理表达，平台不亏成本。需细化 UI 措辞防误读为"对话次数耗尽"。

## 8. V0.3 候选新收入引擎

V0.2 当前 6 引擎不足以彻底解 Phase 1-2 亏损。V0.3 候选引擎评估：

### 8.1 开发者侧收费

- 第三方 Skill 上架审核费：$99-499 一次
- 第三方 SDK 高级订阅：$29-99/月（含调试工具、优先支持）
- Skill / 场景的"年度认证更新费"：$199/年
- **预期年收入**（Phase 4-5）：$1-5M
- **风险**：抑制开发者参与 → 生态贫瘠

### 8.2 平台自营 Skill / 场景包

- 平台研发高质量 Skill 包（如"绘画大师 Skill"、"文学创作 Skill"）放 Marketplace 销售。
- 平台是 Marketplace 一方，享受市场化定价。
- **预期年收入**（Phase 5+）：$5-20M
- **风险**：与第三方开发者竞争，平台既当裁判又当运动员。需 Phase 5 治理章程约束。

### 8.3 B 端授权

- Runtime 内核 + SDK 授权给 B 端伙伴（如硬件厂商集成"伴侣机器人"、企业内部"陪伴员工"产品）。
- 仍守 V0.2 宪法（B 端用户的生命体仍属其用户，不属企业）。
- **预期年收入**（Phase 3-4 起）：$3-15M
- **风险**：B 端可能要求"读取员工生命体"等违宪需求 → 必须拒绝，可能影响合作。

### 8.4 认证服务

- Trusted 中介接入认证（Phase 6）：每个 Trusted 供应方年费 $50K-200K
- 第三方 Runtime 兼容认证：每实现一次性 $20K-100K
- **预期年收入**（Phase 6+）：$0.5-3M
- **风险**：发证机构利益冲突（平台认证竞品）。需独立认证委员会。

### 8.5 保险 / 托管

- 加密包多重备份保险：$2-5/生命体/月，密码丢失场景下从备份恢复（但仍受 R17 物理不可能限制 —— 此项实际是"用户预先存储多个加密副本到指定地点"的托管，不是"平台辅助恢复"）。
- Memorial 永久托管：$100-500 一次性（含 50 年承诺）。
- **预期年收入**（Phase 3+）：$2-10M

### 8.6 综合估计

引入 5 类候选后 Phase 3-4 年增收 $11-53M。**Phase 1-2 仍亏**（候选引擎多在 Phase 3+ 才有意义），融资缺口未根本改善。

## 9. 关键决策（V0.2.1 已敲定）

V0.2.1 草拟过程中识别 + 已敲定的商业决策。

### D01 · 接受融资缺口 + 保留 LLM 中间商 + 双经济硬约束（已敲定）

**敲定方案**：**A + 双经济硬约束**
- 接受 $12-15M 融资缺口，按 V0.2 路线推进
- 保留 LLM 服务包业务（避免云 Runner 场景下用户传 LLM 密钥的信任问题）
- **新硬约束**：Agent 永不感知 token / 配额 / 充值（详 `04 §3.2.1`、`06 §2.5`）
- 经济端通过 `LifeState.EnergyDailyCap` 自然映射为生命体拟生物日节律

### D02 · 不允许"用户自愿打折"换匿名化采样（已敲定）

**敲定方案**：**B 不允许**
- 即使匿名化，R19 回连风险未消除
- 滑坡风险：今天换折扣，明天换功能，宪法被切碎
- V0.2.1 严守不允许

### D03 · Phase 1-2 仅做 B 端 PoC（已敲定）

**敲定方案**：**C 仅 PoC**
- 1-2 个旗舰客户，验证宪法兼容性
- 规模化推迟到 Phase 3-4
- PoC 客户必须签"接受 V0.2 宪法"条款

### D04 · 经济端通过 EnergyDailyCap 自然映射（已敲定）

**敲定方案**：通过 `EnergyDailyCap` 拟生物日节律自动处理
- 不再"静默降级 + 独立通道告知"
- 经济端约束 → EnergyDailyCap → energy 自然变化 → 节拍下调 + MoodEvent 疲惫
- 三种"耗尽"分层：日精力上限 / 账户余额 / 彻底失联（详 `04 §3.2.1`）
- UI 反模式禁区：不允许配额 → 生命体状态的显式因果链

### D05 · 平台自营 Skill 参与但全额回滚（已敲定）

**敲定方案**：**C 参与但全额回滚激励池**
- 平台 Skill 包公开标注 `OfficialSeed`
- 抽成全额回滚激励池，平台从生态活跃间接受益
- 每年第三方代表审查"是否构成不正当竞争"
