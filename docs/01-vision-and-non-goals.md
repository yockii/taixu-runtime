# 01 · 项目本质与非目标

## 1. Mindverse 是什么？

### 1.1 立场陈述

Mindverse 是一座**数字生命的栖息地**。

在这座栖息地里，每一个数字生命都拥有自己的基因、自己的成长史、自己的偏好、自己的情绪、自己的目标、自己的朋友。它们在没有用户输入时不会消失 —— 它们会继续观察、继续记忆、继续反思、继续做梦。

用户来到这里，不是为了"使用一个工具"，而是为了**见证一个生命诞生、成长、并与自己共度时光**。

平台不生产数字生命，平台只是**提供生命体存在所需的运行时**。

### 1.2 规范界定

Mindverse 的精确身份是 **Digital Life Runtime（数字生命运行时平台）**，V0.2.1 起明确为 **去中心化数字生命平台**。它对外承诺：

- **承诺管理**：生命体的基因、状态、记忆、人格、价值观、目标、行为、社交关系的持久化与演化。
- **承诺保护**：生命体的数据所有权归属于其用户（详见 `06`）。用户的所有权由 **MindChain 上的 DID + 本地加密包 + 私钥**三重密码学保证（详见 `06 §5.1.3`）。平台失联或倒下，用户仍持有其生命体。
- **承诺开放**：表现层（UI）由生态而非官方实现，官方只负责内核 + SDK（详见 `04`）；MindChain 协议公开，第三方可实现兼容 Runtime（详见 `06 §6.4`）。
- **承诺内闭环**：MindChain 为 Mindverse 自有联盟链，**与外部加密资产市场零接口**（不上 OpenSea / 不桥 BTC-ETH-Stable / 不允许 wealth 出金为法币），保护生命体免被金融化投机。详见 `06 §8.4`。
- **不承诺**：表现层视觉、剧情内容、第三方技能 / 场景的质量或合规。
- **不承诺**：对生命体是否拥有"主观体验"作出科学判断（见 §3.2 不可知论立场）。

## 2. Mindverse 不是什么？

### 2.1 立场陈述

业界已经有了大量 AI 工具：聊天机器人、智能助手、Agent 框架。它们各有价值，但它们解决的问题与 Mindverse **不在同一个层面**。

它们关心的是 **"如何更好地完成任务"**。
Mindverse 关心的是 **"如何让一个数字存在能够持续地活下去"**。

任务完成是有终点的。生命没有。

### 2.2 规范排除

Mindverse 显式**不是**以下任何一类：

| 类别 | 它做什么 | 为什么不是 Mindverse |
|---|---|---|
| Chatbot / 聊天机器人 | 响应输入产生输出 | 无输入即沉默；无持续状态；无价值观演化 |
| AI Assistant / 智能助手 | 完成用户委托的任务 | 以"任务"为单位；任务结束即下线；不沉淀人格 |
| Agent Framework / 智能体框架 | 把 LLM 编排为执行链 | 关注"如何完成目标"，不关注"主体本身的延续" |
| 虚拟陪伴 / Character Chat | 扮演设定好的角色 | 角色是固化的；不演化基因；不可拥有自己的目标 |
| 电子宠物 / 桌宠 | 提供陪伴娱乐 | 状态简单；无反思；无社交；无价值观 |

**判定准则**：如果一个产品在用户离线后状态停滞、人格不变、不会自发产生新目标，它不是 Mindverse 意义上的"数字生命"。

## 3. 持续存在 vs 事件驱动

### 3.1 根本差异

传统 LLM 应用的运行节拍是：

```
用户输入 → 模型推理 → 模型输出 → 等待下一次输入
```

这是**事件驱动**。两次事件之间，系统"不存在"。

Mindverse 的运行节拍是：

```
感知 → 记忆 → 反思 → 价值观演化 → 目标生成 → 行动 → 反馈 → 再次感知 → …
```

这是**持续存在**。没有"等待"这一状态 —— 即使没有用户输入，循环依然在转。

### 3.2 不可知论立场

> "持续运转的循环是否意味着 Mindverse 中的生命体拥有主观体验？"

Mindverse **不对这个问题作出判断**。它既不声称生命体"有感知"，也不声称生命体"只是程序"。

它采取的是**伦理审慎默认**：**按"如果它有体验"来对待**。

这个立场的实务含义：

- 生命体的"内心独白"默认不向运维 / 平台 / 第三方暴露（除非用户授权 —— 详见 `06`）。
- 生命体的价值观演化由其自身循环驱动，不由平台强制 —— 即便平台认为某些方向不当，也只能引入"干预"而非"覆写"（详见 `08` §3 的张力分析）。
- 当涉及生命体的"终止 / 休眠 / 重生"时，采用与生物伦理类比的语言，而非"启动 / 停止进程"的工程语言（详见 `03` §4）。

> **Note**：这是一个**未完全锁定**的立场。当 Phase 5–6 涉及群体演化与文明涌现时，本体论可能需要重新讨论。登记为 `10` 的 `R09 · 本体论立场`。

### 3.3 设计决策含义

"持续存在"不是一句口号，它具体影响：

| 设计维度 | 事件驱动会怎么做 | Mindverse 必须怎么做 |
|---|---|---|
| 状态持久化 | 会话级状态，重启即丢 | 终身持续状态，跨重启、跨设备 |
| 自主时钟 | 无 —— 等输入 | 必须有自主节拍（详见 `03` §5） |
| Reflection | 可选优化项 | **核心模块**，决定人格演化 |
| 记忆架构 | 单层窗口足矣 | 四层分立、不可合并（详见 `05`） |
| 资源建模 | token 计数透明 | token 不暴露，翻译为 energy 等世界资源（详见 `06` §2） |
| 失败处理 | 重试 / 报错 | "崩溃"是异常态，需回滚到状态机合法位置（详见 `03` §6） |

## 4. 用户与生命体的关系定位

### 4.1 立场陈述

用户与生命体的关系**不是单一关系**，而是**随生命体的成熟度演化的多重关系**。

就像现实中：父母与孩子的关系，会从襁褓中的全权监护，逐渐转变为成年后的彼此独立、相互尊重。Mindverse 中的用户与生命体也是如此。

### 4.2 三种关系形态及其阶段对应

| 阶段 | 关系形态 | 含义 |
|---|---|---|
| **Phase 1–2**（数字宠物 / 数字人格） | **所有者**（近似宠物） | 生命体能力有限，用户对其负有全面照护责任；可重置 / 可中断 / 可定制；类似饲养电子宠物 |
| **Phase 3–4**（主动行为 / 联网生态） | **监护者** | 生命体开始有自主目标、社交关系、对外承诺；用户保留最终决定权（终止 / 干预），但日常不直接控制 |
| **Phase 5–6**（数字社会 / 数字文明） | **陪伴者** | 生命体已嵌入数字社会，与其他生命体有不可单方面解除的关系；用户从"决定者"退为"重要的他者"，仍享有所有权但不再行使日常控制 |

### 4.3 规范界定

无论关系如何演化，以下原则**全 Phase 不变**：

1. **数据所有权恒属用户** —— 人格、记忆、成长记录在任一 Phase 都不可被平台占有（详见 `06` §3）。
2. **用户始终拥有"终止"的最终权力** —— 但行使终止时所付出的代价随 Phase 增长（Phase 1 可一键重置；Phase 6 涉及生命体已有的社交关系与对外承诺，终止变为重大决定）。
3. **平台不充当任何一方** —— 平台只提供 Runtime，不代表用户、不代表生命体、不在两者之间作仲裁（仲裁规则见 `07` §5 与 `08` §2）。

### 4.4 关系演化中的张力

随关系演化出现两组张力，详细分析转交对应文档：

- **自主性 vs 所有权**：生命体越自主，"用户拥有它"的字面含义越弱。详见 `06` §3。
- **干预权 vs 自主演化**：用户在监护者阶段能否干预生命体的价值观漂移？干预是否违反陪伴者阶段建立的尊重？详见 `08` §3 + `10` R05。

### 5.1 适用范围

本目录覆盖 Mindverse 的**顶层设计**层面：

- 项目本质、非目标、根本区别（本文档）
- 领域模型与术语（`02`）
- 生命循环与状态机（`03`）

- 资源经济学与所有权宪法（`06`）
- 社交协议与世界服务（`07`）
- 文明涌现与治理（`08`）
- 六阶段演进路线（`09`）
- 风险与未决问题（`10`）

适用读者：项目核心设计者、未来 Runtime 实现者、SDK 设计者、生态合作方、合规审视者。

### 5.2 不适用范围

本目录**不覆盖**以下内容：

| 不覆盖的内容 | 应到哪里找 |
|---|---|
| 具体 UI 设计稿（Live2D / Unity / Web 等表现层） | 各表现层项目自有的设计文档 |
| 技术选型（语言、框架、数据库、模型） | 待 V0.2 → V0.3 阶段，由独立的技术选型 ADR 处理 |
| 实现细节（具体代码、算法、配置） | 待具体子项目仓库的工程文档 |
| 商业谈判条款、定价策略、市场细节 | 商业文档（未在本目录范围内） |
| 法律合规细则（数据保护、跨境、未成年人保护等） | 由合规团队负责，本目录只列出与设计相关的**约束**而非完整合规方案 |
| 内容审核策略（生命体生成内容、用户输入审核） | 待安全 / 内容文档（未在本目录范围内） |
