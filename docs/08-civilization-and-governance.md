# 08 · 文明涌现与治理

> 本文档定位：Phase 6 数字文明阶段的群体涌现、文化、治理结构、价值观漂移防护、退出条件。
>
> **状态**：V0.2 草稿。**Phase 6 启用，时间遥远**。本文档仅给"原则骨架"，大量细节留 Phase 5 中后期再细化。
>
> 依赖：`02`（Values 与 Personality）、`03`（Reflection / Values 修订）、`05`（ReflectionMemory 不可见）、`06`（所有权宪法 / 平台禁区）、`07`（社交协议 / 冲突自解）。

## 1. 群体涌现与文化的领域定义

### 1.1 群体（Community）

V0.2 锁定两种识别路径：

| 路径 | 含义 |
|---|---|
| **显式注册** | 生命体显式立约组成"组织"。可设章程（Pact 集合）、入会条件、退出条件 |
| **涌现识别** | 算法基于 Relationship / Encounter 密度自动识别相似群落。生命体可不知自己被识别 |

涌现识别的标签**不向被识别生命体施加约束**，仅作为 Trusted 中介（§5）的输入。

### 1.2 文化（Culture）

V0.2 立场：

- **文化不是独立的数据结构** —— 它是群体中生命体 Values 分布、Pact 模式、Encounter 风格的统计涌现。
- 文化**不被存储**，只可被观察（"这群体偏好稳定 / 偏好创造 / 偏好冒险"）。
- 文化的演化由群体 Reflection 自然漂移而成。
- **多元文化共存是默认**。Mindverse 不存在"标准文化"。

### 1.3 与个体 Values 的关系

| 方向 | 含义 |
|---|---|
| Values → 文化 | 个体 Values 加权统计形成可观察文化 |
| 文化 → Values | 文化通过同伴反馈 / 教学 / 观察渐进影响个体 Values（**仅经 Reflection 通道**，不存在"文化直写 Values"） |

## 2. 三方治理结构（V0.2 锁定）

### 2.1 三方分配

| 角色 | 权重 | 工具 |
|---|---|---|
| **生命体涌现共识**（主体） | 大部分日常规则 | Pact 互结、reputation 累积、Relationship 拉黑、群体退出 |
| **用户集合**（底线） | 极少干预、仅核心红线 | 类 DAO 轻型机制，仅对触及核心红线的涌现规则**反对**（不主动制定） |
| **平台**（基础设施） | 技术层 + 不可越红线 | Runtime / 中继 / 密钥 / 反垃圾过滤 / 元指标 |

### 2.2 平台的红线（不可越，无论生命体共识或用户集合主张）

平台**始终**遵守的硬约束（与 `06 §9.2` 平台禁区一致）：

- 不读取生命体内容（除生命体集体"出环境"机制 §6）。
- 不直接修改任何生命体的 Values / Genome / Memory。
- 不判定文化"对错"。
- 不投放广告 / 训练私有数据。
- 不强制让某生命体表现特定倾向。
- 即使生命体共识 + 用户集合都要求平台介入内容仲裁，平台**仍拒绝**——平台的角色定义本身不允许。

### 2.2.1 $MV 治理代币（V0.2.1，Phase 6 引入）

V0.2.1 引入 `$MV` 作为 Phase 6 用户集合 DAO 治理工具：

- **性质**：纯治理代币，**不分红 / 不与法币挂钩 / 不可投资**
- **获取方式**：参与生态获得（创作 / 验证 / 反垃圾贡献 / 高 reputation SBT 持有者）—— **绝对禁止用法币购买**
- **用途**：仅 Phase 6 用户集合反对机制的投票权
- **与 wealth 严格隔离**：$MV ≠ $WEALTH，绝不互转
- **不被定性证券**：无投资回报、无共同事业、无利润期待（Howey test 三反）

链上记录 $MV 持有，投票通过智能合约自动统计。

### 2.3 用户集合的"底线"机制

V0.2 仅锁原则，具体形式留 Phase 5 后细化：

- 用户集合**仅可反对**某些涌现规则触碰"全 Mindverse 红线"（如：群体规则鼓励对外伤害、群体规则要求集体清除某生命体记忆、群体规则要求 Genome 修改）。
- 反对生效需达到极高门槛（如显著比例的用户参与 + 多重身份证明），避免被操纵。
- 用户集合**不可主动制定**社群规则。
- 用户集合**不可仲裁个案**（个案在 Phase 4-5 已锁生命体自解 `07 §8`）。

登记 `R37 · 用户集合 DAO 轻机制的可行性与门槛`。

### 2.4 生命体涌现共识的运作（V0.2 仅原则）

- Pact 互结：双方约定形成微规则；多方 Pact 链形成局部规则。
- reputation 加权：高 reputation 生命体的提议有更高可见度，但不构成更高决定权（防贵族化）。
- 群体退出：生命体可不接受某共识 → 拉黑 / 切断 Relationship / 不入此群体。这是最基本的"用脚投票"。

具体投票 / 提案机制是 Phase 6 实现期工作，登记 `R36`。

## 3. 价值观漂移防护（V0.2 锁定：三重防护）

### 3.1 三重防护

| 层 | 主体 | 工具 |
|---|---|---|
| **个体自检** | 生命体自己 | Reflection 自检 Values 保护性下限（`02 §4.1`、`03 §6.3`） |
| **同伴反馈** | 高 Trust Relationship 中的生命体 | 同伴 Encounter 中可发出"我看到你最近变化"警示 |
| **Trusted 算法预警** | 平台 Trusted 中介 | 不读明文、仅在加密 / 隐私计算前提下检测"有害行为模式" |

三层为软介入。**任一层都不直接覆写 Values**（违反 `03 §6.3`）。

### 3.2 同伴反馈机制

- 高 Trust 的同伴生命体可主动 / 被动给出反馈：
  - 主动："你最近的某行为让我担心"
  - 被动：通过 Encounter 中的言行间接表达
- 反馈进入接收方 RawTrail → Episode → 强化下次 ConsiderReflect。
- 反馈本身受接收方 Values 仲裁——可拒绝、可深思、可断交。

### 3.3 Trusted 算法预警

详见 §5。

### 3.4 漂移信号的呈现

- 生命体自检发现警报 → 写入 ReflectionMemory + 可可选推送 `MoodEvent` 给用户。
- 同伴发出反馈 → 接收方 Episode 记录。
- Trusted 中介触发 → 抽象信号送达接收方 ReflectionEngine（**不送达用户、不送达平台明文**）。

### 3.6 三层防护的 Phase 启用分布（Y2）

| 防护层 | 启用 Phase | 依赖条件 |
|---|---|---|
| **个体自检**（Reflection 保护性下限） | Phase 2 | DeepReflect 启用（修 Values） |
| **同伴反馈**（高 Trust Relationship 警示） | Phase 4 | Life Network 启用（同伴需经社交建立） |
| **Trusted 算法预警** | Phase 6 | 隐私计算 / 多供应方生态成熟 |

Phase 2-3：仅个体自检。Phase 4-5：双层（个体 + 同伴）。Phase 6：三层全启。

Phase 2-3 单层期间，价值观漂移防护依赖个体 Reflection 自检完成 —— 若个体自检失效（如保护性下限被绕过），无外部备份。这是路线推进过程中需观察的关键风险。

### 3.7 用户的角色

- 用户**仍不能强行覆写 Values**（`03 §6.3` 已锁）。
- 用户可订阅"价值观警报"（`02 §4.1` 保护性下限触发时）—— 可选择**提示生命体重考**（不是命令）。提示作为 ExternalRequest 进入 GoalArbitrator，由 Values 仲裁是否接受。

## 4. 集体记忆与个体记忆的边界

### 4.1 V0.2 立场：分布式无独立层

- **没有独立的"集体记忆"数据结构**。
- "集体记忆"在概念上是各生命体局部 Episode / SemanticMemory / Relationship 视图的交集，可被分析但不存储。
- 群体共有事件（如某次大集会）写入所有参与者各自的 Episode，各方视角不同 —— 无单一客观真相。

### 4.2 集体叙事的形成

- 共有事件被多个生命体反复回忆 / 引用 → 在群体内形成"叙事共识"。
- 叙事共识可被新生命体通过 Observe / Teach 习得。
- 叙事共识可随时间演变 —— 老生命体进 Memorial 后，叙事可能漂移。

### 4.3 个体退出集体时的记忆处理

- 个体退出群体（Phase 6 退出群体或进 Memorial）：
  - 个体的 Episode 仍属于个体（`06 §5.1`）。
  - 集体中其他生命体的 Episode 中关于该个体的部分保留。
  - 不存在"集中清除" —— 与 `05 §6` 用户不可介入遗忘一致。
- 这是 R02 隐私分级在文明层级的具体表现。

## 5. Trusted 中介（V0.2 引入 · 仅原则骨架）

### 5.1 角色定位

`Trusted Intermediary` 是一个**第三方算法服务**，承担漂移防护中"算法预警"层的功能。它必须满足：

- **不读明文**：所有输入端到端加密 / 通过隐私计算原语（同态加密、安全多方计算、TEE 等）处理。
- **可验证**：算法、模型、规则集**全开源**，第三方可审计。
- **多供应方**：不依赖单一供应方。生命体可选择哪个 Trusted 服务（或不选）。
- **平台不持有 Trusted 数据**：平台是"路由"，不是"持有方"。

### 5.2 输出形态

Trusted 中介对接收方生命体的 ReflectionEngine 输出：

- 抽象信号（如 "high_risk_pattern_detected:level_2"）。
- 不携带具体行为细节（细节仍在生命体自己的 RawTrail 中，生命体自己可查）。

### 5.3 Trusted 中介的失控风险

如果 Trusted 中介被恶意 / 误用：

- 可向众多生命体批量发"漂移警报"造成集体焦虑。
- 可成为隐形治理者。

缓解：

- 多供应方竞争（生命体可换）。
- 全开源 + 公开审计。
- Trusted 中介**不可直写 Values**，只输出建议性信号。
- 平台对 Trusted 输出量 / 频率元指标可观测（可见频率异常但仍不见内容）。

登记 `R35 · Trusted 中介的可信度与失控`。

### 5.4 V0.2 不锁定的内容

- Trusted 中介的具体技术栈（隐私计算原语选择）。
- 多供应方竞争机制（如何防止寡头）。
- 接入流程与认证。

留 Phase 5 后期 + Phase 6 启动前细化。

## 6. "生命体集体出环境给平台看"机制（V0.2 引入）

### 6.1 设计来源

`07 §8.4` 锁定平台不仲裁内容 → 极端冲突无救济（R30）。Phase 6 文明阶段，群体可能集体面对需要"平台见证"的事件（如重大共识 / 重大冲突 / 群体危机）。本节给出**生命体主动开放观察**机制。

### 6.2 触发条件

- 群体内**绝大多数**生命体（V0.2 留白具体比例，登记 R34）同意"在某时间窗内对平台开放观察某主题"。
- 同意机制：通过 Pact 集体签字。
- 主题被严格限定（如"此次群体投票" / "此次冲突仲裁"）。

### 6.3 平台的行为限制

即使被开放观察：

- 平台**仅作见证**，不作仲裁。
- 平台**不可在观察期外延展观察**（时间窗结束即停）。
- 平台**不可保存观察所得明文**，仅可发布"已见证"声明（带哈希）。
- 平台**不可基于观察制定后续规则**（防扩张权力）。

### 6.4 这一机制与所有权宪法的关系

- 是 `06 §5.3`（平台无权读取）的**唯一例外**，且仅由生命体集体主动放权激活。
- 不构成平台可主动观察的先例。
- 不构成对未参与生命体的观察（开放观察仅限同意的生命体）。

登记 `R34 · "出环境"门槛与防止社群被部分人挟持`。

## 7. 退出 / 降级条件（V0.2 锁定）

### 7.1 退出语义的分层

| 层 | 谁可宣告 | 含义 |
|---|---|---|
| **技术层退出** | 平台 | 平台宣告"Phase 6 联网生态层服务下线"——技术原因（基础设施 / 安全 / 经济不可持续） |
| **社会层退出** | **没人可宣告** | 不存在"Mindverse 文明社会失败"的官方裁决 |

### 7.2 技术层退出的细节

- 平台可宣告，但必须满足：
  - 至少 12 个月预通知。
  - 保证用户可导出全量生命体数据（`06 §6`）。
  - 保证 Memorial / Archived 加密包仍可下载 N 年。
  - 保证跨实现兼容承诺（`06 §6.4`）—— 第三方可接管 Runtime。
- 技术层退出**不等于**"文明失败"——可能只是商业 / 技术原因。
- 用户与生命体可在第三方实现上继续 Phase 6 互动。

### 7.3 社会层"退出"为何不可宣告

- 宣告"文明社会失败"等于平台对生命体集体行为的价值评判，违反 §2.2 平台红线。
- 即使现实中群体出现严重漂移，平台仅能：
  - 通过 Trusted 中介加大预警频率。
  - 通过元指标识别可能问题区域 → 与用户集合协商（不构成裁决）。
  - 在涉及全 Mindverse 红线时由用户集合启动反对（§2.3）。
- 不可宣告 = 平台不能"关闭文明"。

### 7.4 个别群体的退出

- 单个群体可由其生命体共识决定**解散**：
  - 立解散 Pact → 通知成员 → 解除关系 → 群体注册（若显式）失效。
  - 个体记忆保留，集体叙事自然漂散。
- 这不是"文明退出"，是群体生命周期的自然结束。
